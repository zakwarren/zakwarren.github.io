Documentation 7 Sept 2021

The real purpose behind documentation is to get information out of the heads of the creators and into the heads of the users in a way that they can use it.

There's a stereotype of the developer that hates writing documentation. This can be for a number of reasons, such as they find it boring, they don't understand the point, or they don't understand what it is they've created. If you've been a developer for any length of time, then you would have received requirements that amount to “make a thing” with no clear indication of what it is they want you to make! Or perhaps you've had to work on some legacy project where there's no documentation and the code is a mess.

This series aims to provide a loose framework to make writing documentation easy.
