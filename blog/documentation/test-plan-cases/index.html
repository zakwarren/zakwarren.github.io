Test Plan and Test Cases

Documentation 3 Sept 2021

Testing is an essential part of the software development process. Software has bugs, but the more we can catch before deployment the better our software and the happier our users. Test plans and test cases help us do that effectively.

Test Plan

A test plan outlines the strategy of testing software to ensure it meets the design specifications and requirements. It has the following key sections:

Introduction

This section should outline the purpose, project overview, and intended audience.

Test Strategy

This section describes:

  • test objectives – what and why
  • test assumptions
  • test principles (such as the test environment will match the production environment as closely as possible)
  • data approach
  • scope and levels of testing
  • estimated effort and resources

Execution Strategy

The execution strategy section should describe:

  • entry criteria (such as all data loaded) and exit criteria (such as pass rate of 95%)
  • test cycles
  • defect management – labelling, tools, and information required
  • metrics
  • defect tracking and reporting

Test Management Process

This section should describe:

  • test management tool (such as Excel or Jira)
  • test design process
  • test execution process
  • test risks and mitigation factors (such as people, money, schedule, scope, defects)
  • communication plan
  • roles (such as project manager, QA lead, QA team)

Test Environment

The test environment section should include:

  • location of software to be tested
  • permissions
  • hardware and software required for testers
  • contact information for support

Test Cases

Test cases are individual tests that test one specific feature described by a requirement. This means there is a high level of coupling between test cases and requirements.

A test scenario groups several test cases into a logical group, such as a single page of the application. For example, you might have a test scenario to validate a login page. This could include three test cases:

  1. log in with valid credentials
  2. log in with invalid credentials
  3. reset password
